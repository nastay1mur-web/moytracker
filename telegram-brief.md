# Бриф: Telegram Mini App — Трекер привычек
## Пошаговый план разработки v1.0

> Платформа: Telegram Mini App (WebView внутри Telegram-бота)
> Основан на: research.md + экспертной оценке (март 2026)

---

## Общая концепция

Пользователь открывает Telegram-бота → видит список привычек на сегодня → отмечает выполненные → закрывает.
Весь core loop — не дольше 20 секунд.

**Платформа:** Telegram Mini App (WebView)
**SDK:** `window.Telegram.WebApp`
**Язык интерфейса:** Русский
**Стиль:** Нативный Telegram — тема из `themeParams`, минимализм, крупные элементы
**Размер тапа:** минимум 48px

---

## Ключевые отличия от MAX-версии

| Параметр | MAX-версия (заморожена) | Telegram-версия (эта) |
|---|---|---|
| Хранение данных | localStorage | PostgreSQL на бэкенде |
| SDK | Мок + совместим с TG | Нативный `window.Telegram.WebApp` |
| Авторизация | `initDataUnsafe` (без проверки) | `initData` с HMAC-SHA256 верификацией |
| Синхронизация | Нет (только один браузер) | Да (любое устройство) |
| Пуши | Только настройка времени | Реальные сообщения от бота |
| Регистрация бота | Только юрлицо/ИП | Любой через @BotFather |

---

## Архитектура системы

```
Telegram-бот (@BotFather)
    │
    ├─ Команды: /start, /help
    ├─ Кнопка "Открыть трекер" → запускает Mini App
    ├─ Напоминания: cron-задачи → sendMessage() пользователям
    └─ WebApp URL: https://your-app.vercel.app
         │
         └─ Фронтенд (React + Vite)
              │
              ├─ GET /api/habits
              ├─ POST /api/completions
              └─ ...
                   │
                   └─ Бэкенд (FastAPI или Express)
                        │
                        ├─ Верификация initData (HMAC-SHA256)
                        └─ PostgreSQL
                             ├─ users
                             ├─ habits
                             └─ completions
```

## Чего НЕ будет в v1.0

### Функциональность
- Социальные функции: соревнования, лидерборды, совместные вызовы
- Геймификация: очки, достижения, бейджи
- Streak freeze / заморозка серии (v1.1)
- Ретроактивная отметка (отметить за вчера/позавчера)
- Quantity-привычки (выпил 6 из 8 стаканов) — только бинарные
- Детальная аналитика с графиками трендов
- Экспорт данных (CSV)
- Поделиться серией как картинкой (v1.1)
- Напоминания индивидуально на каждую привычку

### Дизайн
- Drag-to-reorder привычек в списке
- Кастомные темы/фоны
- Несколько языков

### Технически
- Оффлайн-режим с синхронизацией
- Web-версия вне Telegram

## Технический стек

| Слой | Технология | Почему |
|------|-----------|--------|
| Фронтенд | Vanilla JS + Vite | Лёгкий бандл, WebView быстрее |
| Стилизация | CSS-переменные из `themeParams` | Нативная тема Telegram |
| SDK | `window.Telegram.WebApp` | Нативный, без обёрток |
| Бот | python-telegram-bot или telegraf | Команды + напоминания |
| Бэкенд | FastAPI (Python) или Express (Node) | REST API + верификация |
| База данных | PostgreSQL | Надёжность, реляции |
| Деплой фронт | Vercel / Netlify | Бесплатный HTTPS (нужен для Mini App) |
| Деплой бэкенд | Railway / Render | Бесплатный тир для MVP |

> **Важно:** Telegram требует HTTPS для Mini App. Vercel / Netlify дают его бесплатно.

### Схема базы данных

```sql
-- Пользователи
CREATE TABLE users (
    id          SERIAL PRIMARY KEY,
    telegram_id BIGINT UNIQUE NOT NULL,
    name        TEXT NOT NULL,
    settings    JSONB DEFAULT '{}',
    created_at  TIMESTAMP DEFAULT NOW()
);

-- Привычки
CREATE TABLE habits (
    id           SERIAL PRIMARY KEY,
    user_id      INTEGER REFERENCES users(id),
    name         TEXT NOT NULL,
    icon         TEXT DEFAULT '⭐',
    color        TEXT DEFAULT '#2AABEE',
    frequency    JSONB DEFAULT '"daily"',  -- 'daily' | ['mon','wed','fri']
    section      TEXT DEFAULT 'any',       -- 'morning' | 'evening' | 'any'
    reminder_time TEXT,                    -- '08:00' | null
    created_at   DATE DEFAULT CURRENT_DATE,
    archived_at  DATE
);

-- Выполнения
CREATE TABLE completions (
    id             SERIAL PRIMARY KEY,
    habit_id       INTEGER REFERENCES habits(id),
    completed_date DATE NOT NULL,
    created_at     TIMESTAMP DEFAULT NOW(),
    UNIQUE(habit_id, completed_date)
);
```
