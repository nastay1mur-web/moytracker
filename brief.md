# Brief: MAX MiniApp — Трекер привычек
## Пошаговый план разработки v1.0

---

## Общая концепция

Пользователь открывает MAX-бота → видит список привычек на сегодня → отмечает выполненные → закрывает.
Весь core loop — не дольше 20 секунд.

**Платформа:** Telegram / MAX Mini App (WebView)
**Язык интерфейса:** Русский
**Стиль:** Нативный MAX — тёмная тема по умолчанию, минимализм, крупные элементы
**Минимальный размер тапабельных элементов:** 48px (выше требования 44px для надёжности)

---

## Архитектура приложения

```
Запуск
  ↓
[Новый?] → ОНБОРДИНГ (3 шага)
[Вернулся] → ГЛАВНЫЙ ЭКРАН
                  ↓
    ┌─────────────┬───────────────┐
    │             │               │
ДЕТАЛИ       ДОБАВИТЬ        НАСТРОЙКИ
ПРИВЫЧКИ     ПРИВЫЧКУ
    │
РЕДАКТИРОВАТЬ
ПРИВЫЧКУ
```

**Bottom navigation (всегда видна):** Сегодня · Прогресс · Настройки

## Экраны

### Экран 0: Загрузка (Splash)

**Что видит пользователь:**
- Логотип / иконка приложения по центру
- Тёмный фон (#1C1C1E или цвет из themeParams)
- Нет текста, нет кнопок

**Действия:** Нет (экран автоматический)

**Переход:**
- Новый пользователь → Онбординг Шаг 1
- Вернувшийся → Главный экран

**Технически:**
- Вызов `Telegram.WebApp.expand()` и `ready()`
- Чтение `initDataUnsafe.user` → запрос к бэкенду
- Длительность: 0.5–1 сек

### Экран 4: Главный экран «Сегодня» ← КЛЮЧЕВОЙ

**Что видит пользователь:**
```
┌─────────────────────────┐
│  Среда, 5 марта    🔥12 │  ← дата + глобальный streak
│  ████████░░  3 из 5     │  ← progress bar + счётчик
│─────────────────────────│
│                         │
│  УТРО                   │  ← секция (если есть утренние)
│  ┌───────────────────┐  │
│  │ 💧  Вода    🔥 7  │  │  ← невыполненная
│  └───────────────────┘  │
│  ┌───────────────────┐  │
│  │ ✓ 📚 Чтение 🔥 3  │  │  ← выполненная (серая, галочка)
│  └───────────────────┘  │
│                         │
│  ВЕЧЕР                  │
│  ┌───────────────────┐  │
│  │ 🏃  Спорт  🔥 12  │  │
│  └───────────────────┘  │
│  ┌───────────────────┐  │
│  │ 🧘  Медитация  —  │  │
│  └───────────────────┘  │
│                         │
│         [  +  ]         │  ← FAB кнопка добавить
│─────────────────────────│
│  Сегодня  Прогресс  ⚙️  │  ← bottom nav
└─────────────────────────┘
```

**Элементы:**
- **Шапка:** дата (16px, серый) + глобальный streak 🔥N (18px bold, оранжевый) — в одну строку
- **Progress bar:** цветная полоска на всю ширину (высота 6px, скруглённая), под ней «X из N» (14px)
- **Секции:** «УТРО» / «ВЕЧЕР» — лейбл 11px uppercase серый, только если есть хоть одна привычка в секции
- **Карточка привычки (невыполненная):**
  - Высота: 60px
  - Левый цветной бордер (4px) — цвет привычки
  - Иконка 28px + название 16px bold
  - Правый край: streak badge (🔥N или просто N, 14px)
  - Фон: #2C2C2E
- **Карточка привычки (выполненная):**
  - Галочка ✓ слева (зелёная, 20px)
  - Текст и иконка серые (opacity 0.5)
  - Перемещается вниз списка
- **FAB «+»:** круглая кнопка 56px, фиксированная позиция над bottom nav, цвет акцента
- **Bottom nav:** 3 иконки с подписями — «Сегодня» / «Прогресс» / «Настройки», высота 56px

**Действия:**
- **Тап по карточке** → отметить выполнено (анимация + вибрация)
- **Тап по выполненной карточке** → снять отметку (вибрация)
- **Долгий тап** → контекстное меню (Редактировать / Пропустить сегодня / Удалить)
- **Тап «+» (FAB)** → экран «Добавить привычку»
- **Тап «Прогресс»** → экран «Прогресс»
- **Тап «Настройки»** → экран «Настройки»
- **Тап на streak 🔥N** → экран «Детали привычки» для самой длинной серии

**Переходы:**
- При отметке последней привычки → экран «Все выполнено» (2 сек) → возврат
- Тап на карточку (не область чекбокса) → Детали привычки

**Состояния главного экрана:**
1. **Пустой (нет привычек)** — иллюстрация + "Добавь первую привычку" + большая кнопка
2. **Есть привычки, ничего не выполнено** — стандартный вид
3. **Частично выполнено** — прогресс бар заполнен частично
4. **Всё выполнено** → триггер экрана-праздника

│                         │
│  Частота                │
│  ┌──────┐ ┌─────────┐   │

**Элементы:**
- `BackButton` Telegram (нативная) — назад без сохранения
- Поле «Название»: text input, высота 52px, автофокус при открытии экрана, placeholder «Например: Пить воду»
- **Выбор иконки:** горизонтальный скролл с иконками (40px каждая), tap = выбрать
- **Выбор цвета:** 7 кружков (28px) — оранжевый, синий, зелёный, фиолетовый, розовый, жёлтый, бирюзовый
- **Частота:** 2 кнопки-переключателя «Каждый день» / «Конкретные дни» (высота 44px)
  - При «Конкретные дни»: появляются 7 кнопок Пн Вт Ср Чт Пт Сб Вс
- **Напоминание:** строка с временем и иконкой → тап открывает time picker
- **Кнопка «Сохранить»:** `MainButton` Telegram — активна как только заполнено «Название»

**Действия:**
- Ввод названия → кнопка «Сохранить» становится активной
- Выбор иконки → подсветка выбранной
- Выбор цвета → цветной кружок с галочкой
- «Конкретные дни» → показать Пн–Вс toggles
- Тап на время → нативный time picker
- `BackButton` / свайп назад → диалог «Отменить изменения?» → Да/Нет
- «Сохранить» (`MainButton`) → записать → вернуться на Главный экран + вибрация success

**Переходы:**
- Успешное сохранение → Главный экран, новая привычка появляется в списке с анимацией

- Toggle «Напоминания» (нативный iOS/Android стиль)
- При toggle ON: строки выбора времени утро/вечер (44px каждая)
- Выбор темы — выбор из 3 вариантов
- Начало недели: Пн / Вс — тап для смены
- «О приложении» → версия + ссылка

**Действия:**
- Toggle уведомлений → включить/выключить пуши от бота
- Тап на время утра/вечера → time picker
- Тап «Тема» → sheet с 3 вариантами
- Тап «Начало недели» → Пн/Вс (мгновенная смена)

## Минимальный технический стек

| Слой | Технология |
|------|-----------|
| Фронтенд | React + Vite |
| Стилизация | CSS Modules + переменные из `themeParams` |
| Навигация | React Router (с ручным BackButton) |
| Telegram SDK | `@twa-dev/sdk` или нативный `window.Telegram.WebApp` |
| Бэкенд | Node.js (Express) или Python (FastAPI) |
| База данных | PostgreSQL — таблицы: `users`, `habits`, `completions` |
| Бот | telegraf (Node) или python-telegram-bot |
| Деплой фронт | Vercel |
| Деплой бэкенд | Railway / Fly.io |

### Минимальная схема БД

```sql
users       (id, telegram_id, name, created_at, settings_json)
habits      (id, user_id, name, icon, color, frequency_json,
             reminder_time, created_at, archived_at)
completions (id, habit_id, completed_date, created_at)
```
